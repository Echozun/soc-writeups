# Network Security Essentials
**Source:** TryHackMe — SOC Level 1 Path **Difficulty:** Easy

## Key Concepts

### Network Assets and Attacker Targets
A network is the organized structure that connects an organization's assets so they can communicate, share resources, and reach the outside world. Each type of asset on that network represents a different kind of target, and the reasons why matter as much as the asset list itself:

- **User workstations (endpoints)** — the PCs and laptops employees use daily, and the most common initial entry point via phishing emails or malicious downloads. Endpoint logs can reveal the malicious process itself, but network logs are frequently what surfaces a C2 connection first.
- **File and database servers** — hold the organization's most valuable asset: the data itself. File servers provide centralized document access; database servers manage structured records like customer, HR, or financial data. This makes them prime ransomware targets (maximizing impact) and the usual destination point for data exfiltration.
- **Application servers (web, email, VPN, etc.)** — externally facing services employees and customers rely on. Because they're internet-facing, they're constantly scanned for vulnerabilities or misconfigurations, and a successful exploit often becomes an attacker's foothold into the internal network.
- **Active Directory / authentication servers** — the identity backbone managing users, groups, computers, and access rights. A prime target for privilege escalation, persistence, and lateral movement.
- **Routers and switches** — network infrastructure connecting networks (routers) and devices within a network (switches). Rarely exposed directly to the internet, but compromise here lets an attacker intercept and manipulate traffic, plant backdoors by rerouting it, or open hidden channels out.
- **Firewalls / perimeter devices** — the primary security gateway inspecting ingress and egress traffic and deciding what's allowed through.

### Network Visibility
Visibility is the ability to actually see what's happening across the network — the foundation for detection, investigation, and overall security posture. It splits into two categories of logs:

- **Host-centric logs** — generated by individual devices: operating system logs (Windows Event Logs, Linux syslog, macOS logs), application logs, and security tool logs (antivirus, EDR, host-based IDS).
- **Network-centric logs** — generated by devices that traffic passes through rather than originates from: firewall logs, IDS/IPS logs, router and switch logs, web proxy logs, and VPN logs.

### The Network Perimeter
The perimeter is the boundary separating an organization's internal network from the external internet — the first and most critical line of defense. In a modern environment, it's no longer just physical edge hardware; it also includes virtual gateways, cloud connections, and remote access points, all of which have to be accounted for when defining where "the edge" actually is.

### Monitoring and Protecting the Perimeter
Because the perimeter is under constant probing, defending it means combining firewalls, IDS/IPS, and access control to inspect traffic and enforce rules. The room walks through a set of worked log examples showing what that looks like in practice:

![Monitoring the Perimeter in Action — port scan and WAF log examples](../../images/network-security-essentials/Pasted%20image%2020260718092432.png)

- **Port scanning** — a firewall log shows one external IP hitting multiple ports on the same internal host in rapid succession. The verdict: a classic port scan, the attacker probing for an open service to target.
- **Web attack detection via WAF** — logs show SQL injection, XSS, and directory traversal attempts against a web server, each tagged with an `attack_type` field and blocked automatically. Filtering for `action=BLOCK` immediately surfaces the threats without having to comb through the allowed traffic, and seeing multiple distinct attack types coming from the same source IP is treated as a high-confidence signal that the site is actively being targeted, not just probed at random.
